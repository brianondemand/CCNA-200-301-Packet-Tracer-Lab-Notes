
# GRE Tunnels (Generic Routing Encapsulation) – CCNA Complete Summary Notes

## What is GRE?

* **GRE (Generic Routing Encapsulation)** is a tunneling protocol developed by Cisco Systems.
* Creates a **virtual point-to-point link** between two routers over an IP network.
* Encapsulates (wraps) one protocol inside another.

---

## Why Use GRE?

GRE allows you to:

* Connect remote sites over the Internet.
* Transport routing protocols across the Internet.
* Carry multicast traffic.
* Carry non-IP protocols (historically).

---

## Basic Idea

Without GRE:

```text
Router A ----- Internet ----- Router B
```

Routing protocols such as:

```text
OSPF
EIGRP
RIPv2 Multicast Updates
```

may not work directly across the Internet.

With GRE:

```text
Router A === GRE Tunnel === Router B
          (Virtual Link)
```

The routers behave as if they are directly connected.

---

## GRE Tunnel Components

### Tunnel Source

The interface where GRE packets originate.

Example:

```text
Router A
Public IP: 100.1.1.1
```

---

### Tunnel Destination

The remote router's public IP.

Example:

```text
Router B
Public IP: 200.1.1.1
```

---

### Tunnel Interface

A virtual interface created on the router.

Example:

```cisco
interface Tunnel0
```

---

## GRE Encapsulation

Original packet:

```text
IP Packet
```

GRE adds:

```text
New IP Header
GRE Header
Original Packet
```

Result:

```text
IP Header
GRE Header
Original Packet
```

This process is called:

```text
Encapsulation
```

## GRE Configuration

### Router A

```cisco
interface Tunnel0
 ip address 10.10.10.1 255.255.255.252
 tunnel source 100.1.1.1
 tunnel destination 200.1.1.1
```

---

### Router B

```cisco
interface Tunnel0
 ip address 10.10.10.2 255.255.255.252
 tunnel source 200.1.1.1
 tunnel destination 100.1.1.1
```

## Routing Over GRE

After the tunnel is established:

### Static Route

```cisco
ip route 192.168.2.0 255.255.255.0 Tunnel0
```

---

### Dynamic Routing

GRE supports:

```text
OSPF
EIGRP
RIP
BGP
```

This is one of GRE's biggest advantages.

---

## GRE and Routing Protocols

### Without GRE

```text
Internet
    ↓
No Multicast Support
```

OSPF and EIGRP updates may fail.

---

### With GRE

```text
OSPF Updates
     ↓
Encapsulated
     ↓
Sent Through Tunnel
```

Routing protocols work normally.

## Advantages of GRE

* Supports dynamic routing protocols.
* Supports multicast traffic.
* Creates virtual point-to-point links.
* Simple to configure.
* Works across the Internet.

---

## Disadvantages of GRE

* No encryption.
* Adds overhead.
* Consumes additional bandwidth.
* Requires another protocol for security.

---

## GRE vs IPsec

| Feature                 | GRE | IPsec             |
| ----------------------- | --- | ----------------- |
| Tunneling               | Yes | Yes               |
| Encryption              | No  | Yes               |
| Dynamic Routing Support | Yes | Limited by itself |
| Security                | Low | High              |

## GRE over IPsec

Common real-world deployment:

```text

 ↓

 ↓

```


