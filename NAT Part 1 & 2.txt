# NAT (Network Address Translation) – CCNA Complete Summary Notes



## What is NAT?

* NAT translates one IP address into another.
* Usually translates **private IP addresses** into **public IP addresses**.
* Performed on a router or firewall.

### Why NAT is Used

* Conserves public IPv4 addresses.
* Allows private networks to access the Internet.
* Hides internal IP addresses from external networks.

---

## Private IPv4 Address Ranges

### Class A


10.0.0.0 - 10.255.255.255


### Class B


172.16.0.0 - 172.31.255.255


### Class C


192.168.0.0 - 192.168.255.255


These addresses are not routable on the Internet.

---

## NAT Terminology

### Inside Local

Private IP address of an internal host.

Example:


192.168.1.10


### Inside Global

Public IP address representing the inside host on the Internet.

Example:


203.0.113.10


### Outside Local

Address of an external host as seen from the internal network.

### Outside Global

Actual public IP address of the external host.

---

## Types of NAT

### 1. Static NAT (One-to-One)

Maps one private IP to one public IP.

Example:


192.168.1.10 ↔ 203.0.113.10


Used for:

* Web servers
* Mail servers
* Devices that must always have the same public IP

---

### 2. Dynamic NAT (Many-to-Many)

Uses a pool of public addresses.

Example:


Private Hosts
     ↓
Pool:
203.0.113.10
203.0.113.11
203.0.113.12


A host gets a public IP from the pool when needed.

---

### 3. PAT (Port Address Translation)

Also called:

NAT Overload


Many private IPs share one public IP.

Example:


192.168.1.10 ┐
192.168.1.11 ├─> 203.0.113.10
192.168.1.12 ┘


PAT differentiates connections using port numbers.

This is the most common NAT type used today.

---

## NAT Example

Before NAT:


PC
192.168.1.10


After NAT:


PC
192.168.1.10
      ↓
Router NAT
      ↓
203.0.113.10


The Internet sees only:


203.0.113.10


---

## Inside and Outside Interfaces

### Configure Inside Interface

```cisco
interface g0/0
 ip nat inside
```

### Configure Outside Interface

```cisco
interface g0/1
 ip nat outside
```

---

## Static NAT Example

```cisco
ip nat inside source static 192.168.1.10 203.0.113.10
```

---

## PAT (NAT Overload) Example

```cisco
access-list 1 permit 192.168.1.0 0.0.0.255

ip nat inside source list 1 interface g0/1 overload
```

The keyword:

```text
overload
```

means PAT.

---

## Verification Commands

Show NAT translations:

```cisco
show ip nat translations
```

Show NAT statistics:

```cisco
show ip nat statistics
```

---

## Advantages of NAT

* Conserves public IPv4 addresses.
* Allows private addressing.
* Hides internal network structure.
* Reduces need for public IPs.

---

## Disadvantages of NAT

* Adds processing overhead.
* Can complicate troubleshooting.
* Some applications may have issues with NAT.

---

## CCNA Exam Tips

### NAT Types

| Type             | Mapping                           |
| ---------------- | --------------------------------- |
| Static NAT       | 1 Private ↔ 1 Public              |
| Dynamic NAT      | Many Private ↔ Pool of Public IPs |
| PAT/NAT Overload | Many Private ↔ 1 Public           |

### Keywords

```text
Static NAT = One-to-One
Dynamic NAT = Pool
PAT = Overload
```

### Remember


Inside Local  = Private IP
Inside Global = Public IP
