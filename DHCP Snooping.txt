# DHCP Snooping – CCNA Complete Summary Notes

## What is DHCP Snooping?

* **DHCP Snooping** is a Layer 2 security feature that protects against **rogue DHCP servers**.
* It filters DHCP messages and allows only trusted DHCP servers to assign IP addresses.
* Builds a database of valid DHCP clients.

---

## Why is DHCP Snooping Needed?

### Problem: Rogue DHCP Server

An attacker connects a fake DHCP server:

```text
PC → Fake DHCP Server
```

The fake server can provide:

* Wrong IP addresses
* Wrong default gateway
* Wrong DNS server

This can lead to:

* Man-in-the-Middle attacks
* Traffic interception
* Denial of Service (DoS)

---

## DHCP Snooping Solution

The switch classifies ports as:

### Trusted Ports

* Allowed to send DHCP server messages.
* Usually connected to:

  * Legitimate DHCP Servers
  * Uplink switches
  * Routers

### Untrusted Ports

* Not allowed to send DHCP server messages.
* Usually connected to end devices.

---

## Trusted vs Untrusted

| Port Type | DHCP Offers Allowed? |
| --------- | -------------------- |
| Trusted   | Yes                  |
| Untrusted | No                   |

Example:

```text
DHCP Server
     │
Trusted Port
     │
Switch
     │
Untrusted Ports
     │
PCs
```

---

## DHCP Snooping Database

The switch builds a binding table containing:

* MAC Address
* IP Address
* VLAN
* Interface
* Lease Time

Example:

```text
MAC                IP Address
0011.2233.4455     192.168.1.10
```

---

## DHCP Message Flow

### DORA Process

```text
Discover
Offer
Request
Acknowledge
```

DHCP Snooping examines these messages.

---

## Enable DHCP Snooping

### Globally

```cisco
ip dhcp snooping
```

---

### Enable for a VLAN

```cisco
ip dhcp snooping vlan 10
```

---

### Configure Trusted Port

```cisco
interface g0/1
 ip dhcp snooping trust
```

Usually the port connected to:

* DHCP Server
* Router
* Distribution Switch

---

## Verify DHCP Snooping

Show snooping status:

```cisco
show ip dhcp snooping
```

Show binding table:

```cisco
show ip dhcp snooping binding
```

---

## DHCP Rate Limiting

Protects against DHCP starvation attacks.

Example:

```cisco
interface fa0/1
 ip dhcp snooping limit rate 10
```

Limits DHCP packets per second.

---

## DHCP Starvation Attack

Attacker sends many DHCP requests:

```text
Attacker
   ↓
Thousands of Requests
   ↓
DHCP Pool Exhausted
```

Result:

```text
Legitimate Users
      ↓
Cannot Get IP Addresses
```

DHCP Snooping rate limiting helps prevent this.

---

## Benefits of DHCP Snooping

* Blocks rogue DHCP servers.
* Prevents DHCP spoofing.
* Prevents DHCP starvation attacks.
* Creates a trusted client database.
* Improves Layer 2 security.

---

## Relationship to Other Security Features

The DHCP Snooping binding table is used by:

### Dynamic ARP Inspection (DAI)

Protects against ARP spoofing.

### IP Source Guard

Prevents IP spoofing.

```text
DHCP Snooping
      ↓
Creates Binding Table
      ↓
DAI and IP Source Guard Use It
```
