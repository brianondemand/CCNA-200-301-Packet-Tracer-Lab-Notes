Network Time Protocol, a fundamental internet standard used to synchronize the clocks of computers and devices over packet-switched, variable-latency networks to within a few milliseconds of Coordinated Universal Time (UTC).



The system clock has several key characteristics:

    Volatile Nature: The system clock retains the current time only while the device is powered on. Once the router reboots, it resets.

    Reset Behavior: If the router does not have a backup hardware clock, it may default to a reference time (00:00 on January 1, 1970) after rebooting.

    Drift: The system clock experiences "clock drift," where the device gradually loses time over a long period. This happens because the software clock depends on the CPU's crystal oscillator, which has minor frequency variations.


NTP constantly measures the time it takes for data to travel back and forth (latency) between your device and a time server, automatically adjusting your clock to account for these delays.

NTP uses a hierarchical system of time sources. At the top of the structure are highly accurate time sources – typically a GPS or atomic clock, providing Coordinated Universal Time (UTC). 

In NTP (Network Time Protocol), stratum is simply a way of showing how far a device is from the original time source.


NTP Stratum Levels


Stratum 0  → Reference clock
             (Atomic clock, GPS clock, etc.)

Stratum 1  → NTP server connected directly
             to a Stratum 0 device

Stratum 2  → Gets time from a Stratum 1 server

Stratum 3  → Gets time from a Stratum 2 server

Stratum 4  → Gets time from a Stratum 3 server




Atomic Clock (Stratum 0)
          │
          ▼
NTP Server A (Stratum 1)
          │
          ▼
Router B (Stratum 2)
          │
          ▼
Switch C (Stratum 3)





NTP Server

NTP uses a client-server architecture; one host is configured as the NTP server and all other hosts on the network are configured as NTP clients.

Simple Network Time Protocol (SNTP)

Simple Network Time Protocol (SNTP) is a simpler NTP version commonly used by small networks. SNTP is a client-only architecture, so it can only receive information from the NTP time servers and cannot provide time for other devices.


Why does a network device need a synchronized clock?

1. Logging and Troubleshooting

2. Security (Authentication and Encryption)

3. Compliance: Many industries have regulatory requirements (such as PCI DSS, HIPAA, or GDPR) that mandate accurate timekeeping for logging and auditing.

4. Time-based Access Lists

5. Network Automation
