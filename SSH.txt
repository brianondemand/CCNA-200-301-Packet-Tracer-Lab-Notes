### SSH (CCNA Summary Notes)

- **SSH (Secure Shell)** is used for **secure remote access** to network devices like routers and switches.

- It replaces **Telnet** because it encrypts all communication.



**Key feature:**

- Provides **encrypted login and session data** (secure management access).

**Default port:**

- TCP 22

---

### How it works

Admin PC → SSH connection → Router/Switch
         (encrypted session)


---

### SSH vs Telnet

| Feature     | SSH       | Telnet     |
| ----------- | --------- | ---------- |
| Security    | Encrypted | Clear text |
| Port        | 22        | 23         |
| Recommended | Yes       | No         |

---

### Basic Cisco Setup (idea-level)

- Set hostname and domain name
- Generate encryption keys
- Create local user
- Enable SSH on VTY lines

Example:

line vty 0 4
 login local
 transport input ssh

---

### CCNA Exam Tips

- SSH is **secure remote management**.
- Uses **TCP port 22**.
- Requires **username + password** login.
- Always preferred over Telnet.

---
