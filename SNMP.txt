**SNMP (Simple Network Management Protocol)**

- Used to **monitor and manage network devices**.

- Allows administrators to check device status remotely.

**Devices Managed:**

- Routers
- Switches
- Servers
- Printers
- Access Points

---

### Components

**SNMP Manager (NMS)**

- Monitors devices.
- Example: Network monitoring server.

**SNMP Agent**

- Software running on the device being monitored.

**MIB (Management Information Base)**

- Database of information about the device.

---

### How It Works


SNMP Manager ↔ SNMP Agent
         ↓
        MIB


The manager requests information from the agent, and the agent provides data from the MIB.

---

### SNMP Messages

- GET → Retrieve information
- SET → Change information
- TRAP → Alert sent by device to manager

---

### Ports

- UDP 161 → SNMP Queries (GET, SET)
- UDP 162 → SNMP Traps

---

### SNMP Versions

| Version | Security                            |
| ------- | ----------------------------------- |
| SNMPv1  | No encryption                       |
| SNMPv2c | No encryption, improved performance |
| SNMPv3  | Authentication and encryption       |


**CCNA Tip:** Use **SNMPv3** because it is the most secure.

---

### SNMP Communities

- A **community string** is like a password used by **SNMPv1** and **SNMPv2c**.

- It controls whether an SNMP manager can access information on a device.
- **Read-Only (RO):** Can view information but cannot make changes.
- **Read-Write (RW):** Can view and modify information.


* Common examples:

  * `public` = Read-Only
  * `private` = Read-Write

- Community strings are **not encrypted**, which is why **SNMPv3** is preferred.

**Remember:**

**Community String = SNMP Password**.



### Exam Reminder

* SNMP monitors and manages network devices.
* Uses **UDP 161** and **UDP 162**.
* Components: **Manager, Agent, MIB**.
* **Trap** = Device sends an alert to the manager.

### One-Line Memory Trick

**SNMP lets network administrators monitor devices and receive alerts when problems occur.**
