# QoS (Quality of Service) – CCNA Complete Summary Notes

## What is QoS?

* **QoS (Quality of Service)** is a set of techniques used to **prioritize important network traffic**.
* Ensures critical applications get the bandwidth and performance they need.
* Commonly used for:

  * Voice (VoIP)
  * Video conferencing
  * Streaming
  * Real-time applications

---

## Why QoS is Needed

Without QoS:

```text
Voice + Video + File Downloads
            ↓
      Compete Equally
            ↓
      Delays and Jitter
```

With QoS:

```text
Voice → Highest Priority
Video → Medium Priority
Data  → Normal Priority
```

---

## Problems QoS Solves

### Latency

* Delay in packet delivery.

### Jitter

* Variation in packet arrival times.

### Packet Loss

* Packets dropped due to congestion.

### Bandwidth Contention

* Multiple devices competing for bandwidth.

---

## QoS Process

### 1. Classification

Identify traffic types.

Example:

```text
Voice
Video
Web Traffic
File Transfers
```

---

### 2. Marking

Traffic is tagged with a priority value.

Common markings:

* DSCP
* CoS

---

### 3. Queuing

Traffic is placed into queues.

Higher-priority traffic is sent first.

Example:

```text
High Priority Queue
   ↓
Voice

Medium Priority Queue
   ↓
Video

Normal Queue
   ↓
Data
```

---

### 4. Congestion Management

Controls how traffic is handled during congestion.

---

## DSCP (Differentiated Services Code Point)

Used to mark packets at Layer 3.

Located in the IP header.

### Common Values

| Traffic Type | DSCP       |
| ------------ | ---------- |
| Best Effort  | 0          |
| Voice        | EF (46)    |
| Video        | AF Classes |

**EF = Expedited Forwarding**

Often used for VoIP traffic.

---

## CoS (Class of Service)

* Layer 2 marking.
* Found in 802.1Q VLAN tags.
* Values range:

```text
0 - 7
```

Higher value = Higher priority.

---

## Trust Boundary

The point where QoS markings are accepted.

Example:

```text
IP Phone → Switch → Router
```

Switch trusts markings from the IP phone.

---

## Queuing

### FIFO (First In First Out)

```text
First Packet In
      ↓
First Packet Out
```

No prioritization.

---

### Priority Queuing

Voice traffic sent before other traffic.

---

## QoS for Voice

Voice requires:

* Low latency
* Low jitter
* Minimal packet loss

Voice traffic is usually:

```text
DSCP EF (46)
```

and receives highest priority.

---

## Cisco QoS Command

Trust QoS markings:

```cisco
mls qos trust dscp
```

---

## Benefits of QoS

* Better voice quality.
* Better video performance.
* Reduced latency and jitter.
* Improved user experience.
* Efficient bandwidth usage.
