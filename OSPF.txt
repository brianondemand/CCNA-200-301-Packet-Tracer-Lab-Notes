## ALGORITHMS

Dynamic routing protocols use algorithms to determine the best path for forwarding packets. The main algorithm types are:



1. Distance Vector: Routers tell neighbors how far destinations are; easy to use but slower and can cause loops.



2. Link State: Routers share a full map of the network; faster and accurate but needs more resources.

When using a link state routing protocol, every router creates a ‘connectivity map’ of the network.

To allow this, each router advertises information about its interfaces (connected networks) to its neighbors. These advertisements are passed along to other routers, until all routers in the network develop the same map of the network.

Each router independently uses this map to calculate the best routes to each destination.

Link state protocols use more resources (CPU) on the router, because more information is shared.

However, link state protocols tend to be faster in reacting to changes in the network than distance vector protocols.


3. Path Vector: Routers share the path through different networks; good for large networks but slower to update.



Stands for Open Shortest Path First

Uses the Shortest Path First algorithm of Dutch computer scientist Edsger Dijkstra. (aka Dijkstra’s algorithm ← remember that name!)

Three versions:
OSPFv1 (1989): OLD, not in use anymore
OSPFv2 (1998): Used for IPv4
OSPFv3 (2008): Used for IPv6 (can also be used for IPv4, but usually v2 is used)

Routers store information about the network in LSAs (Link State Advertisements), which are organized in a structure called the LSDB (Link State Database).

Routers will flood LSAs until all routers in the OSPF area develop the same map of the network (LSDB).


The Link State Database (LSDB) is the collection of Link State Advertisements (LSAs) within an OSPF area, essentially a map of the network topology. 

Every router in the same OSPF area maintains an identical LSDB and uses it with the Dijkstra's SPF algorithm to calculate the best paths to all destinations, ensuring a common, consistent network view and creating a loop-free routing table. 


Each LSA has an aging timer (30min) by default. The LSA will be flooded again after the timer expires.

In OSPF, there are three main steps in the process of sharing LSAs and determining the best route to each destination in the network.


Become neighbors with other routers connected to the same segment.

Exchange LSAs with neighbor routers.

Calculate the best routes to each destination, and insert them into the routing table.


### Areas

An area is a setoff routers and links that share the same LSDB.

The backbone area (area 0) is an area that all other areas must connect to.

Routers with all interfaces in the same area are called internal routers.

Routers with interface in multiple areas are called area border routers (ABRs)

ABRs maintain a separate LSDB for each area they are connected to.

It is recommended that you connect an ABR to a maximum of 2 areas. Connecting an ABR to 3+ areas can overburden the router.

Routers connected to the backbone area (area 0) are called backbone routers.

An intra-area route is a route to a destination inside the same OSPF area.

An interarea route is a route to a destination in a different OSPF area.




OSPF areas divide a large network into smaller, easier-to-manage sections. The backbone, called Area 0, is the central hub that all other areas must connect to. 

This design keeps routing efficient by letting each area handle its own details while sharing only necessary information with the rest of the network. 

For example, a company could use Area 0 for headquarters and separate areas for branch offices, reducing complexity and resource usage.


### Advantages

Dividing a network into OSPF areas makes it easier to manage. Each area only keeps detailed information about itself and exchanges summaries with the backbone. 

This reduces the size of routing tables, lowers CPU and memory usage on routers, and limits how far updates travel when changes occur. 

It also improves stability, since problems in one area don’t easily spread to the entire network.

### Disadvantages

On the downside, OSPF areas make the design more complex. 

Every area must connect to the backbone (Area 0), so planning the topology carefully is necessary. Configuring multiple areas requires more effort and knowledge, and troubleshooting can take longer. 

In smaller networks, the benefits may not outweigh this extra complexity.


### Area Rules

OSPF areas should be contiguous it means that each individual area should be connected not divided up.

All OSPF areas must have at least one ABR connected to the backbone area.

OSPF interface in the same subnet must be in the same area.
