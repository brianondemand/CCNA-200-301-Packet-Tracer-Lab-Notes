# Port Security – CCNA Complete Summary Notes

## What is Port Security?

* **Port Security** is a Layer 2 switch security feature.
* Controls which devices are allowed to connect to a switch port.
* Prevents unauthorized devices from accessing the network.

---

## Why Use Port Security?

* Stops unknown devices from connecting.
* Reduces MAC flooding attacks.
* Increases network security.
* Limits the number of devices per port.

---

## How It Works

The switch learns or is configured with allowed MAC addresses.

Example:

```text
Fa0/1
Allowed MAC: 00:11:22:33:44:55
```

If another device connects:

```text
66:77:88:99:AA:BB
```

A security violation occurs.

---

## Maximum MAC Addresses

Limit the number of devices allowed on a port.

Example:

```cisco
switchport port-security maximum 1
```

Only one MAC address can use the port.

---

## Types of Secure MAC Addresses

### 1. Static Secure MAC

Manually configured.

```cisco
switchport port-security mac-address 0011.2233.4455
```

---

### 2. Dynamic Secure MAC

Learned automatically.

* Stored in RAM.
* Lost after reboot.

---

### 3. Sticky Secure MAC

Learned automatically and added to the running configuration.

```cisco
switchport port-security mac-address sticky
```

Most commonly tested in CCNA.

---

## Violation Modes

### 1. Protect

* Drops unauthorized traffic.
* No log message.
* Port stays up.

```text
Unauthorized Device
        ↓
Traffic Dropped
```

---

### 2. Restrict

* Drops unauthorized traffic.
* Generates logs/counters.
* Port stays up.

```text
Unauthorized Device
        ↓
Traffic Dropped + Log Generated
```

---

### 3. Shutdown (Default)

* Port enters **err-disabled** state.
* Traffic stops completely.
* Most secure option.

```text
Violation
    ↓
Port Disabled
```

---

## Port Security Configuration

### Enable Port Security

```cisco
interface fa0/1
 switchport mode access
 switchport port-security
```

---

### Set Maximum MAC Addresses

```cisco
switchport port-security maximum 1
```

---

### Enable Sticky Learning

```cisco
switchport port-security mac-address sticky
```

---

### Set Violation Mode

```cisco
switchport port-security violation shutdown
```

---

## Verification Commands

Show port security:

```cisco
show port-security
```

Show interface details:

```cisco
show port-security interface fa0/1
```

Show learned MAC addresses:

```cisco
show port-security address
```

---

## Recovering from Err-Disabled

After a shutdown violation:

```cisco
interface fa0/1
 shutdown
 no shutdown
```

---

## Port Security Example

```text
Switch Port Fa0/1

Maximum MACs = 1
Allowed MAC = PC1

PC1 Connects
✓ Allowed

PC2 Connects
✗ Violation
```

---

## Requirements

Port Security works only on:

```text
Access Ports
```

Not on:

```text
Trunk Ports
```

(For CCNA, remember: configure it on access ports.)

---

## CCNA Exam Tips

### Common Commands

```cisco
switchport port-security
switchport port-security maximum 1
switchport port-security mac-address sticky
switchport port-security violation shutdown
```

### Violation Modes

| Mode     | Drops Traffic | Log Message | Port Disabled |
| -------- | ------------- | ----------- | ------------- |
| Protect  | Yes           | No          | No            |
| Restrict | Yes           | Yes         | No            |
| Shutdown | Yes           | Yes         | Yes           |

### MAC Address Types

| Type    | Saved After Reboot        |
| ------- | ------------------------- |
| Dynamic | No                        |
| Static  | Yes                       |
| Sticky  | Yes (after saving config) |

---

## Most Important Exam Facts

```text
Sticky = Learns MAC Automatically
Shutdown = Default Violation Mode
Maximum = Limits Number of Devices
Port Security = Layer 2 Security
```

## One-Line Memory Note

**Port Security limits which MAC addresses can use a switch port and can disable the port if an unauthorized device connects.**
