# FTP & TFTP (Complete CCNA Notes)

## FTP (File Transfer Protocol)

* Used to transfer files between devices.
* Reliable because it uses **TCP**.
* Requires **authentication** (username and password).
* Supports uploading and downloading files.
* Commonly used to back up configurations and transfer IOS images.

**Ports:**

* TCP 21 → Control connection
* TCP 20 → Data connection

**Characteristics:**

* Connection-oriented
* Reliable delivery
* Authentication required
* More overhead than TFTP

---

## TFTP (Trivial File Transfer Protocol)

* Simple file transfer protocol.
* Uses **UDP**.
* No authentication.
* No encryption.
* Primarily used in trusted networks.
* Often used for IOS images, configuration backups, and boot files.

**Port:**

* UDP 69

**Characteristics:**

* Connectionless
* Faster but less reliable
* No authentication
* No directory browsing
* Smaller and simpler than FTP

---

## Cisco Examples

Backup running configuration to TFTP:

```cisco
copy running-config tftp
```

Restore configuration from TFTP:

```cisco
copy tftp running-config
```

Backup running configuration to FTP:

```cisco
copy running-config ftp
```

Copy IOS image from TFTP server:

```cisco
copy tftp flash
```

---

## FTP vs TFTP

| Feature             | FTP    | TFTP  |
| ------------------- | ------ | ----- |
| Transport Protocol  | TCP    | UDP   |
| Port                | 20, 21 | 69    |
| Authentication      | Yes    | No    |
| Reliability         | High   | Lower |
| Connection-Oriented | Yes    | No    |
| Encryption          | No     | No    |
| Complexity          | Higher | Lower |

---

## CCNA Exam Tips

* **FTP = TCP = Reliable = Authentication**
* **TFTP = UDP = Simple = No Authentication**
* TFTP is commonly used by Cisco devices for configuration and IOS transfers.
* FTP uses separate control and data connections.
* Neither FTP nor TFTP encrypts data.

### Memory Trick

```text
FTP  = Full-featured, Password, TCP
TFTP = Tiny FTP, No Password, UDP
```
