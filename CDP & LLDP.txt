CDP (Cisco Discovery Protocol) and LLDP (Link Layer Discovery Protocol) are Layer 2 neighbor discovery protocols. 

They broadcast device information (such as hostname, IP, hardware platform, and port) to directly connected devices, allowing for dynamic topology mapping and troubleshooting.




## Cisco Discovery Protocol (CDP) 

Cisco Proprietary

Cisco devices only

Enabled by default on Cisco devices

Multicast MAC 01:00:0C:CC:CC:CC

Uses Cisco-specific TLVs (Type-Length-Value)



##COMMANDS


cdp run / no cdp run: Enable or disable CDP globally.


cdp enable / no cdp enable: Enable or disable CDP on a specific interface.


show cdp: View global CDP status and timers.


show cdp neighbors: Displays directly connected Cisco devices, the interface they connect to, platform, and port capabilities.


show cdp neighbors detail: Provides deeper, granular info (e.g., neighbor's management IP, OS version, VLAN information).






## Link Layer Discovery Protocol (LLDP) 

Industry Standard (IEEE 802.1AB)

Multi-vendor (Cisco, Juniper, HP, etc.)

Disabled by default on Cisco devices

Multicast MAC 01:80:C2:00:00:0E

Uses LLDP-MED for VoIP auto-configuration 





## COMMANDS

lldp run / no lldp run: Enable or disable LLDP globally.

lldp transmit / lldp receive: Enable LLDP per interface (these are often enabled by default if global lldp run is active).

show lldp: View global LLDP settings and timers.


show lldp neighbors: Displays directly connected devices and basic connection details.


show lldp neighbors detail: Provides detailed OS version and device capabilities.
